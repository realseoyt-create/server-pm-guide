# 서버 PM 및 Failover 테스트 가이드

> 본 문서는 서버 PM(Planned Maintenance) 작업 및 Failover 테스트를 처음 수행하는 담당자도 이 문서만 보고 안전하게 진행할 수 있도록 작성되었습니다.

## 대상 환경

| 서버 | OS | 역할 | 수량 |
|------|-----|------|------|
| HP-UX 서버 | HP-UX B.11.31 | 애플리케이션 서버 (Active/Standby) | 2대 |
| AIX 서버 | AIX 5.3 | 백업 서버 | 1대 |
| Web 서버 | Windows Server | 웹 서버 (IIS) | 2대 |

## 서비스 구성

- **ORACLE DB**: 데이터베이스 (HP-UX Active 서버에서 운영)
- **JAVA WAS**: 애플리케이션 서버 (WebLogic/JBoss 등)
- **RV (TIBCO Rendezvous)**: 미들웨어 메시지 데몬
- **IIS**: 웹 서비스 (Windows)
- **Cron**: 배치 작업 스케줄러

## 전체 작업 흐름 요약

```
D-7  사전 점검 수행 → 이상 항목 정리
D-5  부서원 공지 메일 발송 (크론탭 확인 요청 포함)
D-3  CAB 회의 자료 제출 → 승인 획득
D-1  Start/Stop 스크립트 최종 검토 / 롤백 기준 확정
D-0  체크리스트 기반 작업 수행 → Failover 테스트 → 검증 → 보고서 작성
```

---

## ⚠️ 절대 놓치면 안 되는 핵심 3가지

> 🔴 **1. 작업 전 반드시 백업 최종 확인** — 백업 없이 진행하다 장애 발생 시 복구 불가
>
> 🔴 **2. Cron 일시 중지** — 작업 중 배치가 돌면서 오작동/데이터 오염 발생 가능
>
> 🔴 **3. ORACLE Listener 기동 확인** — DB를 올려도 Listener 없으면 앱에서 접속 불가
