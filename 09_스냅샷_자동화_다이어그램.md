# 09. 점검 자동화 — pre_snapshot + post_compare

> **핵심**: 사람이 기억에 의존해 수동으로 비교하던 것을
> 스크립트 2번 실행으로 대체한다.

---

## 1. 워크플로우 비교

```mermaid
%%{init: {'theme': 'neutral', 'themeVariables': {'fontSize': '14px'}}}%%
flowchart LR

    subgraph OLD [" 기존 — 수동 방식 "]
        direction TB
        o1([작업 전\n12:30])
        o2[ps -ef\n프로세스 조회]
        o3[ss -tlnp\n포트 확인]
        o4[crontab -l\nCron 확인]
        o5[df -h\n디스크 확인]
        o6[결과 수동 기록\n메모장 / 텍스트]
        o7([Failover 실행])
        o8[프로세스 재조회\n눈으로 비교]
        o9[포트·디스크 재확인]
        o10{차이 발견?}
        o11[원인 파악\n수동 조치]
        o12([완료])

        o1 --> o2 --> o3 --> o4 --> o5 --> o6
        o6 --> o7
        o7 --> o8 --> o9 --> o10
        o10 -- 있음 --> o11 --> o12
        o10 -- 없음 --> o12
    end

    subgraph NEW [" 자동화 — pre + post "]
        direction TB
        n1([작업 전\n12:30])
        n2["pre_snapshot.sh\n한 번 실행"]
        n3["10개 항목 자동 수집\n/tmp/snapshot_before/ 저장"]
        n4([Failover 실행])
        n5["post_compare.sh\n한 번 실행"]
        n6["자동 diff 비교\n전·후 항목 대조"]
        n7["리포트 출력\n✅ / ⚠️ / ❌"]
        n8([완료])

        n1 --> n2 --> n3
        n3 --> n4
        n4 --> n5 --> n6 --> n7 --> n8
    end
```

## 2. 자동 수집·비교 항목

```mermaid
%%{init: {'theme': 'neutral'}}%%
flowchart LR

    PRE["pre_snapshot.sh\n작업 전 1회 실행"]

    PRE --> A["① 프로세스 목록\nps -ef"]
    PRE --> B["② Listen 포트\nss -tlnp"]
    PRE --> C["③ 디스크 사용률\ndf -h"]
    PRE --> D["④ 메모리 · CPU\nfree -m / top"]
    PRE --> E["⑤ VIP 위치\nip addr show"]
    PRE --> F["⑥ Crontab 전 계정\ncrontab -l"]
    PRE --> G["⑦ Oracle 상태\nV$INSTANCE"]
    PRE --> H["⑧ Listener 상태\nlsnrctl status"]
    PRE --> I["⑨ 시스템 로그\njournalctl / messages"]
    PRE --> J["⑩ OS 정보\nhostname / uptime"]

    A & B & C & D & E & F & G & H & I & J --> STORE["/tmp/snapshot_before/\n작업 전 상태 보존"]

    STORE --> POST["post_compare.sh\n작업 후 1회 실행"]

    POST --> R1["✅ 정상\n변화 없음"]
    POST --> R2["⚠️ 경고\n확인 필요"]
    POST --> R3["❌ 오류\n즉시 조치"]

    R1 & R2 & R3 --> RPT["리포트 저장\n/tmp/compare_report_*.txt"]
```

## 3. 비교 항목별 판정 방식

```mermaid
%%{init: {'theme': 'neutral'}}%%
flowchart TD
    subgraph COMPARE [" post_compare.sh 판정 로직 "]
        direction TB

        C1["프로세스 변화\ndiff before/after ps"]
        C2["포트 1621\nss grep 결과"]
        C3["Crontab 내용\ndiff before/after"]
        C4["VIP 위치\nip addr 비교"]
        C5["Oracle STATUS\nV$INSTANCE"]
        C6["PJOSCD 등록\nlsnrctl status"]
        C7["DataGuard lag\nV$DATAGUARD_STATS"]
        C8["디스크 사용률\ndf -h 증감"]

        C1 --> P1{변화?}
        C2 --> P2{LISTEN?}
        C3 --> P3{diff?}
        C4 --> P4{위치 일치?}
        C5 --> P5{OPEN?}
        C6 --> P6{등록됨?}
        C7 --> P7{lag = 0?}
        C8 --> P8{증가폭?}

        P1 -- 없음 --> OK1[✅]
        P1 -- 있음 --> WA1[⚠️]
        P2 -- 열림 --> OK2[✅]
        P2 -- 닫힘 --> ER2[❌]
        P3 -- 동일 --> OK3[✅]
        P3 -- 다름 --> ER3[❌]
        P4 -- 일치 --> OK4[✅]
        P4 -- 다름 --> WA4[⚠️]
        P5 -- OPEN --> OK5[✅]
        P5 -- 아님 --> ER5[❌]
        P6 -- 등록 --> OK6[✅]
        P6 -- 미등록 --> ER6[❌]
        P7 -- 0초 --> OK7[✅]
        P7 -- 지연 --> WA7[⚠️]
        P8 -- 5%미만 --> OK8[✅]
        P8 -- 5%이상 --> WA8[⚠️]
    end
```

## 4. 효과 비교

| 항목 | 수동 방식 | pre + post 자동화 |
|------|-----------|-------------------|
| 작업 전 소요 시간 | 15 ~ 30분 | 1분 (스크립트 실행) |
| 작업 후 소요 시간 | 20 ~ 40분 | 1분 (스크립트 실행) |
| 비교 항목 수 | 담당자마다 다름 | 매번 동일한 10개 항목 |
| 비교 방법 | 기억 / 메모 의존 | diff 기반 정량 비교 |
| 누락 가능성 | 높음 | 없음 |
| 결과 기록 | 수동 작성 | 자동 파일 저장 |
| 판정 기준 | 주관적 | ✅ / ⚠️ / ❌ 명확 |
| 재현성 | 담당자에 따라 편차 | 누가 실행해도 동일 |
| 보고서 활용 | 별도 정리 필요 | 리포트 파일 바로 첨부 |

## 5. 실행 한 눈에 보기

```
  [D-Day 12:30]

    bash pre_snapshot.sh
    └─ 10개 항목 수집 완료
       저장: /tmp/snapshot_before/

  [Failover 실행]

  [완료 후]

    bash post_compare.sh
    └─ 전·후 자동 비교
       ✅ 정상: 7   ⚠️ 경고: 1   ❌ 오류: 0
       저장: /tmp/compare_report_20250713_1620.txt
```
