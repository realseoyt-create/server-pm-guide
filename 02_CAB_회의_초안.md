## 1. 작업 개요

### 1-1. 작업 목적

- 서버 정기 PM(Planned Maintenance) 수행: 시스템 안정성 유지 및 이상 유무 점검
- Failover 테스트 수행: PJTSAP/PJTSEC 간 VIP 전환 정상 동작 검증
- Oracle 19C, RVD 기동/중지 절차 표준화 및 스크립트 검증

### 1-2. 작업 대상 시스템

| 서버명 | VIP | Real IP | OS | 역할 |
|--------|-----|---------|-----|------|
| PJTSAP | 12.230.210.207 | 12.230.210.205, .206 | RHEL 8.10 (64bit) | 애플리케이션 서버 |
| PJTSEC | 12.230.210.203 | 12.230.210.201, .202 | RHEL 8.10 (64bit) | 애플리케이션 서버 |

### 1-3. 대상 서비스 및 미들웨어

| 서비스 | 현황 | 비고 |
|--------|------|------|
| Oracle 19C DB | 운영 중 | Service: PJOSCD, Port: 1621 |
| RVD (TIBCO Rendezvous Daemon) | 운영 중 | 양 서버 |
| Cron 배치 | 운영 중 | 각 계정 등록 |
| DB 동기화 | 운영 중 | PJTSAP ↔ PJTSEC |

## 2. 작업 상세 내용

### 2-1. 작업 단계 및 소요 시간

| 단계 | 작업 내용 | 예상 소요 |
|------|-----------|-----------|
| 1 | 사전 스냅샷 수집 및 최종 점검 | 30분 |
| 2 | Cron 백업/중단, 모니터링 중단, DB 동기화 중단 | 20분 |
| 3 | RVD 중지 (양 서버) | 10분 |
| 4 | Oracle DB 중지 (양 서버) | 20분 |
| 5 | Failover 테스트 (VIP 전환 확인) | 30분 |
| 6 | Oracle DB 기동 + Listener 확인 | 20분 |
| 7 | RVD 기동 | 10분 |
| 8 | 복구 확인 (DB 동기화 재개, Cron 활성화) | 20분 |
| 9 | 서비스 정상 동작 검증 + 전후 비교 리포트 | 30분 |
| 버퍼 | 이슈 대응 | 30분 |
| **합계** | | **약 3.5시간** |

### 2-2. 서비스 중지/기동 순서

```
[중지 순서]
Cron 중단 → 모니터링 중단 → DB 동기화 중단
  → RVD 중지 → Oracle Listener 중지 → Oracle DB Shutdown

[기동 순서 - 역순]
Oracle DB Startup → Oracle Listener 기동 → 서비스 등록 확인(PJOSCD/1621)
  → RVD 기동 → Cron 재활성화 → 모니터링 재개 → DB 동기화 재개
```

## 4. 리스크 분석 및 완화 방안

| 리스크 | 가능성 | 영향도 | 완화 방안 |
|--------|--------|--------|-----------|
| Failover 미동작 | 중 | 높음 | 수동 VIP 전환 절차 사전 준비 |
| Oracle 기동 실패 | 낮 | 높음 | Alert Log 사전 점검, DBA 대기 |
| Listener 서비스 미등록 (PJOSCD) | 중 | 높음 | 기동 후 lsnrctl status 반드시 확인, ALTER SYSTEM REGISTER 준비 |
| Cron 미복구 | 중 | 중 | 백업본 비교 체크리스트 포함 |
| DB 동기화 과도한 lag | 중 | 중 | 작업 시간 단축 노력, 허용 lag 사전 확인 |
| RVD 기동 순서 오류 | 중 | 중 | Oracle 기동 완료 확인 후 RVD 기동 순서 준수 |

## 5. 롤백 계획

### 롤백 결정 기준
- 작업 시작 후 **[X]시간** 이내 서비스 정상화 안 될 경우
- 롤백 결정권자: [이름/직책]

### 롤백 절차
```
1. 롤백 결정 선언 및 공지
2. 현재 신규 Active 서버에서 서비스 중지
3. 구 Active 서버 재기동
4. 구 Active 서버에서 서비스 기동 (Oracle → RVD 순서)
5. VIP 원복 확인
6. 서비스 검증 후 완료 공지
```

## 6. 사전/사후 확인 항목

### 사전 확인
- [ ] 백업 최신 완료 확인

- [ ] Tablespace 여유 > 15%
- [ ] Cron 등록 현황 파악 완료
