### 1-1. 작업 목적

- Failover 테스트 수행: HP-UX Active/Standby 전환 동작 정상 여부 검증
- 서비스 재기동 절차 표준화 및 Start/Stop 스크립트 검증

## 2. 작업 상세 내용

### 2-1. 작업 단계 및 소요 시간

| 단계 | 작업 내용 | 예상 소요 시간 |
|------|-----------|----------------|
| 1 | 작업 전 최종 시스템 상태 점검 | 30분 |
| 2 | Cron 일시 중지 | 5분 |
| 3 | Web 서버(IIS) 서비스 중지 | 10분 |
| 4 | JAVA WAS 중지 | 10분 |
| 5 | RV 데몬 중지 | 5분 |
| 6 | ORACLE DB 중지 | 15분 |
| 7 | HP-UX Active 서버 중지 | 10분 |
| 8 | Failover 전환 확인 (Standby → Active) | 20분 |
| 9 | ORACLE DB 기동 | 15분 |
| 10 | RV 데몬 기동 | 5분 |
| 11 | JAVA WAS 기동 | 15분 |
| 12 | Web 서버(IIS) 서비스 기동 | 10분 |
| 13 | 서비스 정상 동작 검증 | 30분 |
| 14 | Cron 재등록 및 확인 | 10분 |
| 15 | 이상 없을 시 작업 완료 선언 | - |
| **합계** | | **약 3시간** |

> 💡 **버퍼 포함**: 이슈 발생 시 대응 시간을 고려하여 **총 4시간** 확보 권장.

### 2-2. 작업 절차 요약

```
[중지 순서]
Web 서버 (IIS 중지) 
  → JAVA WAS 중지 
  → RV 데몬 중지 
  → ORACLE DB shutdown 
  → Cron 중지 
  → HP-UX Active 서버 전원 Off (또는 네트워크 차단)

[Failover 확인]
HP-UX Standby 서버가 Active로 전환되었는지 확인
VIP(Virtual IP) 전환 확인

[기동 순서]
ORACLE DB startup 
  → Listener 기동 확인 
  → RV 데몬 기동 
  → JAVA WAS 기동 
  → Web 서버 (IIS 기동) 
  → Cron 재활성화 
  → 서비스 검증
```

## 4. 리스크 분석 및 완화 방안

| 리스크 | 가능성 | 영향도 | 완화 방안 |
|--------|--------|--------|-----------|
| Failover 자동 전환 미동작 | 중 | 높음 | 수동 전환 절차 사전 준비, cmruncl 명령 사전 확인 |
| ORACLE DB 기동 실패 | 낮음 | 높음 | Alert Log 사전 확인, DBA 대기 |
| Cron 미복구 | 중 | 중 | 작업 후 체크리스트에 Cron 확인 항목 포함 |
| Rollback 시간 초과 | 낮음 | 높음 | 롤백 기준 시간 사전 확정 (예: 작업 시작 후 2시간) |
| IIS 서비스 기동 실패 | 낮음 | 중 | IIS 재설치 없이 재기동 방법 사전 확인 |
| 작업 중 배치 실행 | 중 | 중 | 작업 전 Cron 일시 중지 필수 |

## 5. 롤백 계획

### 5-1. 롤백 결정 기준

- 작업 시작 후 **[X시간]** 이내에 서비스 정상화가 안 될 경우 롤백 결정
- Failover 전환 후 DB 기동 실패 시 원래 Active 서버 재기동 검토
- 롤백 결정권자: [이름/직책]

### 5-2. 롤백 절차

```
1. 롤백 결정 선언
2. 현재 신규 Active 서버(구 Standby)에서 서비스 중지
3. 구 Active 서버(원본) 재기동
4. 구 Active 서버에서 서비스 순서대로 기동
5. 서비스 정상 확인
6. 원인 분석 후 재작업 일정 협의
```
