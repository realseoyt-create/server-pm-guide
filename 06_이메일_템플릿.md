작업 시간 동안 서비스가 중단되오니 미리 일정을 확인하시어 업무에 차질이 없도록 준비 부탁드립니다.

  - 예상 소요 시간: 약 [X]시간

■ 작업 내용
  1. 서버 정기 PM 점검
  2. HP-UX Active/Standby Failover 테스트
  3. 서비스 재기동 절차 검증

■ 서비스 영향 범위
  - 영향 서비스: [서비스명 목록]
  - 영향 시간: 작업 전체 시간 동안 서비스 접근 불가

■ 대상 서버

## 메일 2: Cron 등록 여부 확인 요청 (D-7)

> 💡 **대상**: 서버에 cron을 등록하여 배치를 운영하는 팀/담당자

```
제목: [요청] [작업일] 서버 PM 작업 관련 Cron 등록 현황 확인 요청

수신: [각 담당자별 개별 발송 또는 관련 팀 전체]
발신: [본인 이름]

안녕하세요, [본인 이름]입니다.

[작업일] 서버 PM 작업과 관련하여 협조 요청 드립니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 요청 배경
서버 PM 작업 시간([시작시간] ~ [종료시간]) 동안 서버가 중지됩니다.
이 시간에 Cron으로 등록된 배치 작업이 실행되면 오류가 발생할 수 있습니다.
사전에 배치 실행 일정을 파악하여 충돌을 방지하고자 합니다.

■ 확인 요청 사항
아래 내용을 [회신 기한: 작업일 D-3]까지 회신 부탁드립니다.

  ① 귀 팀/업무에서 아래 서버에 등록된 Cron이 있습니까?
     - 서버: [HP-UX 서버 hostname 또는 IP]
     - 계정: [확인이 필요한 계정 목록]

  ② Cron이 있다면 아래 정보를 알려주세요:
     - 실행 계정:
     - 실행 시간 (분/시/일/월/요일):
     - 실행 스크립트/명령:
     - 작업 중요도 (작업 시간 중 실행되면 재실행 가능한지 여부):

  ③ Cron이 없다면: "해당 없음" 으로 회신 부탁드립니다.

■ 작업 시간 중 배치 처리 방침 (참고)
  - 작업 시간 전에 해당 시간대 Cron은 일시 중지 예정
  - 작업 완료 후 즉시 재활성화
  - 중요 배치가 있을 경우 작업 전 수동 실행 또는 작업 후 재실행 협의 가능
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

빠른 회신 부탁드립니다. 감사합니다.
[본인 이름] 드림
[연락처]
```

## 메일 3: 연계 시스템 담당자 협조 요청 (D-5)

```
제목: [협조 요청] [작업일] 서버 PM 작업 관련 연계 서비스 중단 안내

수신: [연계 시스템 담당자]
발신: [본인 이름]

안녕하세요, [본인 이름]입니다.

[작업일] 서버 PM 작업과 관련하여 연계 서비스 중단에 대한 협조를 요청드립니다.

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
■ 중단 서비스 정보
  - 서비스명: [우리 서비스명]
  - 중단 시간: [작업 시작] ~ [작업 종료]
  - 영향 API/인터페이스: [연계 인터페이스명]

■ 협조 요청 사항
  1. 위 시간 동안 [우리 서비스]로 향하는 연계 요청을 중지하거나
     오류 처리가 가능한 상태로 준비해 주시기 바랍니다.
  2. 작업 완료 후 저희 측에서 완료 공지를 드릴 예정이며,
     해당 공지 수신 후 연계를 재개해 주시면 됩니다.
  3. 작업 시간 중 미처리된 건수가 있을 경우 재처리 방안을 함께 협의해 주시기 바랍니다.

■ 재처리 방안 (사전 협의 필요)
  - [ ] 작업 시간 중 요청은 폐기
  - [ ] 재시도 로직이 있어 자동 처리
  - [ ] 수동 재처리 필요 → 담당자 간 협의 후 처리
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

문의 사항이 있으시면 [연락처]로 연락 주시기 바랍니다.
감사합니다.
[본인 이름] 드림
```
