# 08. Failover 대응 방식 개선 — AS-IS vs TO-BE

> **핵심 변화**: 사후 발견(Reactive) → 사전 예방(Proactive) + 🤖 LLM Crontab 품질 점검

---

## AS-IS vs TO-BE 프로세스 비교

```mermaid
flowchart TD
    subgraph ASIS["❌  AS-IS — 사후 대응 방식"]
        direction TB
        a1["📋 프로세스 스냅샷\n(ps -ef 저장)"]
        a2["📋 Crontab 백업\n(저장만 해둠)"]
        a1 --> fo1
        a2 --> fo1
        fo1(["⚡ Failover 실행"])
        fo1 --> a3{"❓ 미기동\n프로세스 발견?"}
        a3 -- "있음 ⚠️" --> a4["🔍 원인 조사\n수동 조치 / 장애 대응"]
        a4 --> a5["📝 사후 보고"]
        a3 -- "없음" --> a5

        style ASIS fill:#fff0f0,stroke:#cc3333
        style fo1 fill:#ffbbbb,stroke:#cc0000,color:#111
        style a3 fill:#ffdddd,stroke:#cc3333
        style a4 fill:#ffcccc,stroke:#cc3333
    end

    subgraph TOBE["✅  TO-BE — 선제 대응 방식"]
        direction TB
        b1["📋 프로세스 스냅샷\n(ps -ef 저장)"]
        b2["📋 Crontab 수집\n(전 계정)"]
        b1 --> cmp
        b2 --> cmp
        cmp(["🔄 교차 비교\nprocess_cron_compare.sh"])
        cmp --> b3{"Crontab 없는\n프로세스 존재?\n= 고아 프로세스"}
        cmp --> llm["🤖 LLM 분석\nCrontab 전체 품질 점검\n① 실행 스크립트 존재 여부\n② 중복 실행 방지 누락 (flock)\n③ 로그 미기록 탐지\n④ 실행 시간 충돌\n⑤ 오류 알림 미설정"]
        b3 -- "있음\n→ 고아 프로세스 목록" --> b4
        b3 -- "없음" --> b4
        llm -- "개선 가이드 제공" --> b4
        b4["✅ 선제 조치\n고아 프로세스 처리\n+ Crontab 개선 반영"]
        b4 --> fo2
        fo2(["⚡ Failover 실행\n이미 준비 완료"])
        fo2 --> b5["✅ 빠른 검증"]
        b5 --> b6["📝 보고 / 이슈 최소화"]

        style TOBE fill:#f0fff0,stroke:#339933
        style cmp fill:#cce5ff,stroke:#3366cc,color:#111
        style llm fill:#e8e0ff,stroke:#6644cc,color:#111
        style fo2 fill:#aaddaa,stroke:#339900,color:#111
        style b3 fill:#ddffdd,stroke:#339933
        style b4 fill:#ccffcc,stroke:#339933
    end
```

---

## 무엇이 달라지는가

| | AS-IS | TO-BE |
|--|-------|-------|
| **비교 시점** | Failover **이후** | Failover **이전** |
| **고아 프로세스** | Failover 후 발견 | 사전 탐지·조치 |
| **Crontab 품질** | 확인 안 함 | 🤖 LLM이 자동 점검 |
| **문제 발견** | 장애 발생 후 | 작업 전 인지 |
| **담당자 확인** | Failover 중 (긴박) | Failover 전 (여유) |
| **대응 방식** | 긴급 수동 조치 | 선제 계획 수립 |

## 고아 프로세스란?
